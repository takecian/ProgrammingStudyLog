符号化と

それと理論は

一緒でも



慣れないとしんどいねな話。


メールの話。

その前に小噺として符号化の話。

パソコンの世界には 0 と 1 しかありません。

例えば「A」の文字も PC でみれば 0100 0001 という 0 と 1 の羅列なわけで。
この列を PC は「A」という文字として表示(符号化されてる)してるだけで。

＃厳密には文字コードによって列の値は異なる。
＃この１つの 0 とか 1 とかをビットと呼ぶ。
＃「A」の文字は 8 ビットを使って表しているという。


そんな 0 と 1 の世界でこんな問題が出たとする。

一週間の天気を 0 と 1 の列を使って示しなさい。
ビット列は短ければ短いほどよい。

天気は「晴れ」、「曇り」、「雨」、「雪」の４種類だとすると
普通に考えると

晴れ：00
曇り：01
　雨：10
　雪：11

と一日の天気を表すのに 2 ビット必要。
つまり一週間に14ビット必要。


でも、
例えば 70% の確率で晴れ、20% が曇り、5% で雨、5% で雪という地方だとすると
他にもっといい方法があるのではないかと。

晴れ：1
曇り：01
　雨：001
　雪：000

こうすると一週間の天気を並べてもちゃんと天気を識別できる。
しかも一日の天気を表すのに、

（晴れ）＋（曇り）＋（雨）＋（雪）
1×70% ＋ 2×20% ＋ 3×5% ＋ 3×5% ＝ 1.4

という計算から平均 1.4 ビットあれば表現できると言える。

つまり一週間は 9.8 ビット、およそ 10ビットでいけますよ、と。


これは符号化について学ぶ時に最初に聞く話だけど、
似たようなことが現実世界で。


ほとんどの人はメールを打つとき「かな入力」方式かと。

「お」を打つときには１を５回押す、普通の方式。

それともうひとつ、「ツータッチ」方式。

「お」を打つときには１を押して、５を押す、いわゆるポケベル方式。


今日メール打つのめっちゃはええなって人がいて
見たらツータッチ方式で入力してた。

ポケベル方式は
一文字を基本２タッチで打つ。

かな入力方式は
一文字を１タッチ〜５タッチまで。


英語では e とか i をよく打つって話を聞いたけど
日本語ではどんなもんなんだろう。

その辺の情報は手元にないけど
とりあえず自分の携帯の文字入力をポケベル方式にしてみた。


符号化とかいう以前に打つ速度が急激に低下。
効率化を図る前に慣れることが必要ですな。


というわけで携帯でメールうつのがめんどいので以下私信。
